Can you explain how Python's asyncio event loop works under the hood?
A

Sure! At a high level, Python's asyncio event loop is a single-threaded scheduler that manages coroutines, I/O callbacks, and timed events.

Here's the core cycle:

  • Select — the loop polls the OS for I/O readiness (epoll on Linux, kqueue on macOS, IOCP on Windows)
  • Dispatch — ready callbacks are run in order
  • Step — coroutines that were awaiting that I/O get resumed

A simplified version looks like this:

while tasks:
    ready = selector.select(timeout)
    for key, mask in ready:
        callback = key.data
        callback()
    for coro in scheduled:
        coro.send(None)

The key insight is that everything runs in one thread — concurrency comes from voluntarily yielding control with await, not from OS threads.
